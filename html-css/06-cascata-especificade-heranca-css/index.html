Cascata, especificidade e herança em CSS

Falaremos agora sobre 3 conceitos importantes em css: cascata, especificidade e herança!

  • Cascata: para seletores que possuem o mesmo nível de primeira camada (na mesma camada), em caso de existir duas regras para um mesmo elemento, prevalecerá quele que aparecer por ultimo no documento .css. Este li possui 2 regras no arquivo css, mas apenas a ultima é funcional.
  • Especificidade: Elementos que especificam mais qual objeto a regra css deve ser aplicado, tem mais prioridade. Por exemplo: um seletor de classe é mais específico que um seletor de tipo(tag), pois seleciona um numer menor de elementos no html. Portanto, a classe será priorizada, mesmo que o seletor tipo seja inserido por ultimo na código css.
  • Heranca: Um elemento filho em css herda de um elemento pai algumas propiedades. Por exemplo: color e font-size. No entanto, nem todas as propiedades funcionam desta maneira.

    Por exemplo: isso é um paragrafo filho de uma div que está dentro de um li, mas mesmo assim o css é aplicado a ele.

Maneiras de Controlar a herança de um atributo CSS
Atributo Descrição Execução
inherit Estabelece a mesma estilização do elemento pai
Essa div tem cor da letra por padrao azul Essa classe span também terá a cor azul
initial Estabelece a estilização inicial do objeto para o padrao inicial
Essa div tem cor da letra por padrao azul Essa classe span terá a cor preta, que é a cor inicial (sem ser as pre-definidas pelo navegador)
revert Estabelece a estilização inicial do objeto para os estilos padrao do navegador
Essa div tem cor da letra por padrao azul Essa tag á tera cor azul pois é o padrao do navegador
unset Estabelece a estilização inicial seguindo a que tipo de herança ele tinha (inherit, initial).
Essa div tem cor da letra por padrao azul Essa classe span terá a azul, pois seu padrao é azul e inherit
